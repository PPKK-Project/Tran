# 규칙
## 🚀 I. 소통 및 협업 (Communication & Collaboration)
팀의 원활한 소통과 진척도 관리를 위한 규칙입니다.
### 1. 📅 정기 회의 및 진척도 공유

* **주간 회의:** 매주 1회 대면 또는 비대면으로 회의를 진행합니다. 
* **일일 스탠드업:** 매일 **10분 내외**로 비대면 음성 채팅 등을 통해 다음 세 가지를 간결하게 공유합니다. 
    * 어제 한 일 (진척도)
    * 오늘 할 일 (계획)
    * 현재 겪는 문제/장애물 (블로커)
* **불참 시 사전 고지:** 개인적인 사정으로 회의나 작업 참여가 어려울 경우, **반드시 미리 팀원에게 이야기**해야 합니다. 

---

## 🧠 II. 문제 해결 및 학습 (Problem Solving & Learning)

AI 사용에 대한 명확한 기준을 설정하여 팀원들의 학습과 책임감을 강화합니다.

### 2. 🤖 AI 사용 및 이해 책임

* **AI 활용:** AI(ChatGPT, Copilot 등) 사용은 가능합니다.
* **AI 코드 이해:** AI가 생성하거나 추천한 코드나 라이브러리를 사용할 경우, **그 코드를 사용하는 이유**와 **작동 방식**을 스스로 이해하고 **다른 팀원에게 설명이 가능**하도록 학습해야 합니다. 
* **자립적 문제 해결 우선:** 문제나 에러 발생 시, AI에 의존하기 전에 **먼저 스스로 해결을 시도**하거나 **팀원들에게 질문**하여 함께 해결하는 것을 우선합니다. 
* **AI 사용 시 책임:** AI를 사용하여 에러를 해결한 경우에도, 해당 해결 방식을 **팀원에게 논리적으로 설명**할 수 있어야 합니다.

## 💾 III. 코드 관리 및 품질 (Code & Git Management)

코드의 일관성과 Git 저장소의 안정성을 확보하기 위한 규칙입니다.

### 3. 🎯 Git 브랜치/커밋 명명 규칙 통일

* **커밋 메시지 형식:** `[타입]: 내용` 형식으로 통일합니다. 
    * **예시:** `feat: 로그인 기능 구현`, `fix: 회원가입 에러 수정`
* **브랜치 명 형식:** `[타입]/짧은-기능명` 형식으로 통일합니다. 
    * **예시:** `feature/user-login`, `bugfix/cart-error`

### 4. 🔄 Git 작업 안정성 확보

* **작업 전 Pull 필수:** 새 작업을 시작하거나 푸시하기 전에는 **항상** `git pull`을 실행하여 원격 저장소의 최신 코드를 로컬에 반영해야 합니다. **(충돌 방지)** 

### 5. 🔒 민감 파일 공유 규칙

* **설정 파일 공유:** `.env`나 `application.properties`와 같이 환경 설정 정보가 담긴 파일은 **수정될 때마다** 팀 채팅방(예: 디스코드)에 최신화하여 공유합니다.

**✅ 커밋 메시지는 내 코드를 대신 설명하는 '기록'입니다.**
